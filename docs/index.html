Tradernet API tools in Codex, over Model Context Protocol with per-tool approvals.

Connects Codex to the Tradernet API through Model Context Protocol over Streamable HTTP. Exposes tools for quotes, historical candles, market references, news, portfolio reads, orders, reports, tariffs, watchlists, alerts and SID-based session utilities.

Real account. Tradernet MCP operates against a real Tradernet account. Orders, tariffs, uploads, watchlist edits, alerts and account-switching actions can affect real account state.

Requirements

  1. A Tradernet account with API access.
  2. A Tradernet API key pair from tradernet.com/tradernet-api/auth-api.
  3. Codex with plugin and MCP support.
  4. Environment variables for credentials.

Secrets (env only)

Do not put secrets in the repo or plugin files. Use environment variables in ~/.config/tn-mcp/credentials.env (chmod 600):

export TN_API_KEY="your-api-key"
export TN_API_SECRET="your-api-secret"
export TN_LOGIN="user@example.com"
export TN_PASSWORD="your-password"

SID cache

TN_SID is optional — most HMAC tools use TN_API_KEY and TN_API_SECRET. For Codex-local SID caching use ~/.codex/tn-mcp/session.env (mode 600):

export TN_SID="session-id"

MCP configuration

The plugin ships this Codex MCP server config. It contains only environment variable names — never credentials.

{
  "mcpServers": {
    "TN": {
      "url": "https://tradernet.com/mcp/tn",
      "env_http_headers": {
        "X-TN-API-Key": "TN_API_KEY",
        "X-TN-API-Secret": "TN_API_SECRET",
        "X-TN-SID": "TN_SID"
      }
    }
  }
}

Write/trade tools are pinned to approval_mode: "prompt" in .mcp.json, while the default mode stays auto for read-only tools.

Safety model

Read-only tools can run when requested:

  • quotes, historical candles, news, market status
  • portfolio reads, report reads, tariff reads

Write or trade tools require explicit confirmation (approval_mode: "prompt"):

  • placing or canceling orders, stop-loss
  • changing tariffs, uploading files
  • editing alerts and watchlists
  • switching connected users
  • login or SMS authentication

Data notes

  • Numbers often come back as strings ("p": "10.08600000") — cast before math.
  • quotes_get_hloc candle order is [HIGH, LOW, OPEN, CLOSE]; the ticker is a string ("AAPL.US"), not an array.
  • Large responses — use pagination and filters (max_count, date ranges).
